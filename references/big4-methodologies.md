## Deloitte · LEAP Dashboard

### 核心方法论

```
┌─────────────────────────────────────────────────────┐
│              Deloitte TNFD Service Line             │
├─────────────────────────────────────────────────────┤
│                                                     │
│  1. Diagnostic → 现状评估 + 差距分析                │
│  2. Data Integration → 多源数据整合                 │
│  3. LEAP Dashboard → 可视化风险分析                 │
│  4. Disclosure → 报告生成 + 审计准备                │
│  5. Assurance → 第三方鉴证（可选）                  │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 核心工具
- **LEAP Dashboard**：内部开发的地理空间分析平台
- **Data Orchestrator**：多源数据整合工具（WDPA、ENCORE、内部数据）
- **Risk Visualizer**：交互式风险地图

### 差异化优势
- ✅ 技术驱动：自有 Dashboard 平台
- ✅ 可视化强：交互式地图 + BI 仪表盘
- ✅ 端到端：从评估到审计一站式

### 局限性
- ❌ 价格高：中小企业负担不起
- ❌ 黑盒：Dashboard 不对外销售
- ❌ 依赖数据团队：需要客户有数据能力

```
┌─────────────────────────────────────────────────────┐
│           PwC "Five Things You Should Know"         │
├─────────────────────────────────────────────────────┤
│                                                     │
│  1. TNFD 是什么 → 框架概览 + 商业案例               │
│  2. 谁需要披露 → 适用范围 + 监管要求                │
│  3. 如何准备 → LEAP 四阶段执行指南                  │
│  4. 数据从哪来 → 数据源清单 + 质量评估              │
│  5. 下一步行动 → 路线图 + 资源规划                  │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 核心工具
- **TNFD Readiness Assessment**：准备度评估问卷
- **Disclosure Template**：披露模板（对标 14 项建议）
- **Materiality Matrix**：重要性评估矩阵

### 差异化优势
- ✅ 结构化强：5 步框架清晰易执行
- ✅ 适合初学者：首次披露企业的友好入口
- ✅ 审计衔接：与 PwC 审计服务无缝对接

### 局限性
- ❌ 量化弱：财务风险量化不够深入
- ❌ 模板化：定制化程度低
- ❌ 审计绑定：倾向于引导到 PwC 审计

安永 CCaSS（Climate Change and Sustainability Services）是安永全球整合审计、税务、财务交易和咨询优势的团队。

**核心能力**：
- 气候风险与脱碳战略
- ESG 披露与报告编制
- 可持续鉴证（全球统一方法论）
- TNFD LEAP 评估

### 核心方法论

```
┌─────────────────────────────────────────────────────┐
│           安永 CCaSS TNFD 服务流程                   │
├─────────────────────────────────────────────────────┤
│                                                     │
│  1. 差距分析 → 对标 TNFD 14 项建议                    │
│  2. 数据梳理 → 利用现有运营数据 + 价值链信息         │
│  3. LEAP 评估 → 四阶段完整评估                       │
│  4. 目标设定 → 结合 SBTN 五大步骤                    │
│  5. 报告编制 → 治理 + 战略 + 风险 + 指标四大支柱     │

```
┌─────────────────────────────────────────────────────┐
│           KPMG NATURE Framework                     │
├─────────────────────────────────────────────────────┤
│                                                     │
│  N - Navigate → 战略导航 + 治理设计                 │

## 共同模式（剥削大模型的关键）

**四大收几百万，底层逻辑都是这 5 步**：

```
1. 诊断（Diagnostic/Readiness/Navigate）
   → 你现在在哪？差距是什么？
   
2. 评估（Assessment/Identify/Assess）
   → 风险有多大？哪些是重要的？
   
3. 量化（Quantification/Value at Risk）
   → 财务影响是多少？
   
4. 披露（Disclosure/Report）
   → 报告怎么写？覆盖 14 项建议吗？
   
5. 验证（Assurance/Evaluate）
   → 数据可信吗？审计能过吗？
```

**TNFD Skill 的 Prompt 设计要映射这 5 步**：

| Prompt 文件 | 对应阶段 | 四大映射 |
|------------|---------|---------|
| `01-locate.md` | 诊断 | Deloitte Diagnostic / PwC Readiness |
| `02-evaluate.md` | 评估 | KPMG Assess / EY Identify |
| `03-assess.md` | 量化 | EY Quantify / Deloitte Risk Visualizer |
| `04-prepare.md` | 披露 | PwC Disclosure / KPMG Report |
| `05-assurance.md` | 验证 | 四大 Assurance 通用 |

## 审计视角 · 红旗清单

**四大审计师一定会查的 6 个红旗信号**：

| 红旗 | 检查方法 | 常见于 |
|------|---------|--------|
| 🚩 供应链缺失 | 检查披露边界是否包含 Scope 3 | 首次披露企业 |
| 🚩 数据未验证 | 检查数据来源是否有第三方背书 | 自报数据 |
| 🚩 量化缺失 | 检查是否有财务影响数字 | 定性描述为主 |
| 🚩 目标无依据 | 检查目标是否参考 SBTN 等标准 | 拍脑袋目标 |
| 🚩 治理未文档化 | 检查董事会会议纪要 | 家族企业 |
| 🚩 计算方法变更 | 检查往年披露对比 | 频繁变更方法 |

**TNFD Skill 的 `05-assurance.md` 要用这个清单**。

## 使用建议（Skill 集成）

### Prompt 设计原则

1. **诊断阶段**（`01-locate.md`）：
   - 用 PwC Readiness Assessment 风格
   - 输出：准备度评分 + 差距清单

2. **评估阶段**（`02-evaluate.md`）：
   - 用 KPMG Assess 风格
   - 输出：重要性矩阵 + 优先级排序

3. **量化阶段**（`03-assess.md`）：
   - 用 EY VaR 风格（简化版）
   - 输出：财务影响范围（低/中/高）

4. **披露阶段**（`04-prepare.md`）：
   - 用 PwC Disclosure Template 风格
   - 输出：14 项建议覆盖检查

5. **验证阶段**（`05-assurance.md`）：
   - 用四大审计红旗清单
   - 输出：红旗信号 + 改进建议

### 话术风格

| 场景 | 话术 |
|------|------|
| 指出缺失 | 「根据四大审计标准，这个披露缺少量化依据，审计师会出保留意见」 |
| 推荐数据 | 「ENCORE 是 TNFD 官方推荐数据源，四大都在用」 |
| 提醒风险 | 「供应链风险未覆盖是常见红旗信号，建议补充 Scope 3 评估」 |
| 引导行动 | 「下一步建议：1. 完成资产坐标收集 2. 运行 ENCORE 匹配 3. 生成初步报告」 |
